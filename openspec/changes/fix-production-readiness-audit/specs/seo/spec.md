## MODIFIED Requirements

### Requirement: OG Image Configuration
The system SHALL reference existing OG image files in metadata configuration.

#### Scenario: Default OG image
- **WHEN** no custom OG image is specified
- **THEN** the default OG image SHALL be `/images/og-image.jpg`
- **AND** the image file SHALL exist in the public directory

## ADDED Requirements

### Requirement: OG Image Format Compliance
The system SHALL provide Open Graph images in formats supported by major social platforms (JPG, PNG, WebP).

#### Scenario: Social media share preview
- **WHEN** a page URL is shared on Facebook, LinkedIn, or Twitter
- **THEN** the OG image SHALL be rendered correctly
- **AND** the image format SHALL be JPG or PNG (not SVG)
- **AND** the image dimensions SHALL be 1200x630 pixels
- **AND** the file size SHALL be less than 200KB

### Requirement: Single Source of Truth for SEO Configuration
The system SHALL use only App Router conventions (`sitemap.ts`, `robots.ts`) for SEO file generation, avoiding duplicate configuration sources.

#### Scenario: Sitemap generation
- **WHEN** the sitemap is requested at `/sitemap.xml`
- **THEN** it SHALL be generated by `src/app/sitemap.ts`
- **AND** no conflicting `next-sitemap.config.js` SHALL exist

#### Scenario: Robots.txt generation
- **WHEN** robots.txt is requested at `/robots.txt`
- **THEN** it SHALL be generated by `src/app/robots.ts`
- **AND** no static `public/robots.txt` SHALL exist

### Requirement: Apple Touch Icon
The system SHALL provide an Apple Touch Icon for iOS bookmark functionality.

#### Scenario: iOS home screen bookmark
- **WHEN** a user adds the site to their iOS home screen
- **THEN** the bookmark SHALL display a 180x180px branded icon
- **AND** the icon SHALL be located at `/apple-touch-icon.png`
